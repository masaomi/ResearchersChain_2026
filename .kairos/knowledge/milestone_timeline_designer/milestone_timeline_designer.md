# Milestone Timeline Designer

## Purpose
Create realistic, trackable milestones for research projects.

## Milestone Categories

| Phase | Typical Milestones |
|-------|-------------------|
| Planning | Literature review complete, hypothesis formalized, ethics approved |
| Data | Data acquired, QC passed, preprocessing complete |
| Analysis | Primary analysis complete, sensitivity analysis done |
| Validation | Results replicated, peer feedback incorporated |
| Writing | Draft complete, co-author review, submission |
| Revision | Reviewer response, revision submitted, accepted |

## Timeline Estimation Rules
- Add 50% buffer to initial estimates (planning fallacy)
- Identify critical path (longest sequential dependency chain)
- Mark hard deadlines (grant submission, conference, graduation)
- Plan for iteration (first analysis rarely final)

## Gantt Chart Elements
```
Month:        1    2    3    4    5    6
Literature:   ████
Data prep:    ░░░░████
Analysis:          ░░░░████████
Validation:                  ████
Writing:                     ░░░░████
Submission:                       ████
```
(████ = active, ░░░░ = preparation/overlap)

## Checkpoint Review
At each milestone, ask:
- Are we on track?
- Has scope changed?
- Do risks need re-evaluation?
