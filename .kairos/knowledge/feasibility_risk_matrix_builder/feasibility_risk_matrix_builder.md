# Feasibility and Risk Matrix Builder

## Purpose
Systematically evaluate whether a proposed project is achievable and identify risks.

## Feasibility Dimensions

| Dimension | Questions to Ask |
|-----------|-----------------|
| Data | Is data available? Sufficient quality/quantity? Access permissions? |
| Methods | Are tools/methods established? Expertise available? |
| Time | Can it be completed within the deadline? |
| Budget | Are computational/experimental costs covered? |
| Personnel | Are required skills available in the team? |
| Ethics | Can ethical approval be obtained in time? |

## Risk Matrix

| | Low Impact | High Impact |
|--|-----------|-------------|
| **High Probability** | Monitor | Mitigate immediately |
| **Low Probability** | Accept | Prepare contingency |

## Mitigation Strategies
- **Technical risk**: Pilot experiment, alternative methods identified
- **Data risk**: Backup datasets, synthetic data for development
- **Timeline risk**: Prioritized milestones, minimum viable analysis
- **Personnel risk**: Cross-training, documented protocols

## Output
A feasibility score (Go / Go with conditions / No-go) with documented risks and mitigations.
