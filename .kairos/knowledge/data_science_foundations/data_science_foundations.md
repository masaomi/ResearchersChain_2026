# Data Science Foundations

## Core Disciplines
- **Statistics**: Inference, hypothesis testing, regression, Bayesian methods
- **Machine Learning**: Supervised, unsupervised, reinforcement learning
- **Data Engineering**: ETL, data pipelines, storage, quality assurance
- **Visualization**: Exploratory and explanatory data visualization

## ML Model Categories
| Category | Examples | Use When |
|----------|---------|----------|
| Linear models | Linear/logistic regression, GLM | Interpretability needed, linear relationships |
| Tree-based | Random Forest, XGBoost, LightGBM | Tabular data, non-linear, feature importance |
| Deep learning | CNN, RNN, Transformer | Images, sequences, large unstructured data |
| Dimensionality reduction | PCA, t-SNE, UMAP | High-dimensional data exploration |
| Clustering | k-means, DBSCAN, hierarchical | Unsupervised grouping |

## Data Science Workflow
1. Problem definition and hypothesis formulation
2. Data collection and understanding
3. Exploratory Data Analysis (EDA)
4. Data cleaning and preprocessing
5. Feature engineering
6. Model selection and training
7. Evaluation and validation
8. Interpretation and communication
9. Deployment and monitoring

## Key Principles
- **No free lunch**: No single model is best for all problems
- **Bias-variance tradeoff**: Balance underfitting vs overfitting
- **Garbage in, garbage out**: Data quality determines result quality
- **Occam's razor**: Prefer simpler models when performance is comparable
- **Reproducibility**: Version data, code, models, and environments

## Common Pitfalls
- Data leakage (training on test information)
- Selection bias in data collection
- Confounding variables
- p-hacking and multiple comparisons
- Overfitting to noise
- Simpson's paradox
